# 命运和弦 —「个人健康教练闭环」改造方案

## 一、现状审计

### 现有模块（8 个导航标签）
| 模块 | 现有内容 | 核心问题 |
|------|---------|---------|
| 📊 仪表板 | 打卡/热量收支/喝水/日历/心情/习惯 | ❌ 只是信息堆积，没有"今日任务/今日建议" |
| 🍽️ 饮食 | 700+食物库/搜索/自定义 | ❌ 没有快捷记录、重复上次、套餐 |
| 💪 运动 | 150+种/9分类/计时器/呼吸 | ❌ 没有难度/器械/场景筛选 |
| 🎯 计划 | 5套训练计划 | ❌ 没有首次问卷自动生成，不能调整 |
| 📏 身体 | 体重/围度/BMI/成就/睡眠/PR | ⚠️ 功能可用但未联动分析 |
| ✓ 任务 | CRUD | ⚠️ 与饮食/运动/身体无关联 |
| 📈 分析 | 图表+周报 | ❌ 无完成率/问题/风险/建议 |
| ⚙️ 设置 | 档案/目标/导出/Firebase | ❌ 档案字段太少，缺登录体验 |

### 核心缺失
1. **首次问卷** — 不知道用户目标/偏好/限制
2. **今日闭环** — 用户打开 APP 不知道该做什么
3. **跨模块联动** — BMR/TDEE 没影响饮食目标，训练完成度没反馈
4. **智能分析** — 图表不"说话"
5. **快捷记录** — 每次都要选半天
6. **计划可调** — 生成了不能改

## 三、本次实现（P0）详细设计

### 3.1 首次问卷 Wizard — OnboardingWizard.jsx

```
页面1: 选择目标       [减脂] [增肌] [塑形] [保持] [健康]
页面2: 输入身体信息   年龄/身高/体重/性别
页面3: 活动水平       频率 1-7天/周 | 单次时长 15-90分钟
页面4: 饮食偏好       [普通] [素食] [高蛋白] [低碳水]
页面5: 可用器材       [无器械] [哑铃] [弹力带] [健身房] [全都有]
页面6: 身体限制       无/膝盖/腰/肩/其他
```

填写完成后自动：
- 生成 7 天训练计划（匹配目标+器材+频率）
- 设置热量目标 = TDEE ± 500（减脂/增肌）
- 设置蛋白目标 = 体重×1.6-2.2g
- 保存到 localStorage + 跳转到首页

### 3.2 今日首页 — TodayPage.jsx

```
┌─ 今日打卡状态 ──────────────────┐
│ 🔥 打卡 + 连续天数               │
│ 📊 热量进度: 已摄入/目标 + 百分比 │
│ 💧 喝水进度: 杯数/8杯            │
├─ ✅ 今日待办 ──────────────────┤
│ [☐] 记录早餐                    │
│ [☐] 完成今日训练: 胸+三头         │
│ [☐] 记录体重                     │
│ [☐] 喝够8杯水                    │
│ [☐] 记录心情                     │
├─ 💡 今日建议 ──────────────────┤
│ → 蛋白质还差40g，建议加餐鸡蛋     │
│ → 今日训练强度较低，可加15分钟有氧  │
│ → 已连续打卡5天，继续加油！        │
└────────────────────────────────┘
```

### 3.3 快捷记录

- **最近食物**: 显示最近用的 8-10 个食物，一键添加
- **我的套餐**: 保存常用组合（早餐ABC、练后餐）
- **复制昨日**: 一键复制昨天的饮食记录
- **常用运动**: 保存常用训练模板
